# Реальные данные покрытия — как собрать и залить

Покрытие («какой провайдер заводит интернет в этот дом») — главный актив платформы.
Готового API по РФ нет, поэтому данные собираются вручную/из партнёрских фидов. Ниже — формат
и практический план.

## Формат CSV

Шаблон: [coverage-template.csv](coverage-template.csv). Первая строка — заголовок.

| Колонка | Обяз. | Что это |
|---|---|---|
| `city` | да | Город (напр. `Казань`) |
| `street` | да | Улица как пишется (`улица Баумана`) |
| `house` | да | Номер дома (`10`, `10к2`) |
| `provider` | да | Слаг провайдера латиницей (`rostelecom`, `domru`, `mts`) — по нему группируются записи |
| `provider_name` | нет | Отображаемое имя (`Ростелеком`). Если провайдера ещё нет — создастся с этим именем |
| `tech` | нет | Технология: `GPON` / `FTTB` / `ADSL` |
| `payout` | нет | Ставка CPA за подтверждённое подключение, ₽ (для расчёта выручки в админке) |
| `source` | нет | Метка источника фида (`rostelecom-admitad`) — видно в /admin/coverage, помогает следить за свежестью |

Повторный импорт **не создаёт дублей** и **освежает** существующие записи (обновляет `updatedAt`/`source`).

## Как достать данные (реалистичный план)

1. **Сначала — деньги, потом данные.** Зарегистрируйся в CPA-сети (Admitad / Pampadu) и возьми
   офферы тех провайдеров, которых реально можешь монетизировать. Заводи в покрытие **только их** —
   иначе заявки некуда «продавать».
2. **Зона покрытия по каждому провайдеру:**
   - если партнёрка даёт фид покрытия — выгрузи его и приведи к формату выше;
   - если нет — у каждого провайдера на сайте есть «проверить адрес». Проходишь по домам района
     и фиксируешь, кто доступен. Нудно, но это и есть ров.
3. **Начни узко.** Один город → один район → 20–50 домов → 2–3 провайдера. Проверь весь путь на
   реальном адресе: поиск → сравнение → заявка → звонок → подключение → выплата.

## Как залить

**Через админку:** `/admin/coverage` → «Импорт покрытия из CSV» → выбрать файл.

**Через CLI** (для больших фидов/cron):

```bash
ADMIN_PASSWORD=<пароль> SITE_URL=https://tarify.rumedia-cdn.com \
  npm run import-coverage -- data/coverage-template.csv rostelecom-admitad
```

## Когда данные станут реальными

- Убери демо-данные: сейчас в проде засеяны 3 демо-города. Реальный импорт **добавляется** к ним,
  поэтому демо-города надо удалить (через Neon/Prisma Studio) перед публичным запуском.
- Выключи демо-плашку: в Vercel выставь `NEXT_PUBLIC_DEMO=0` и передеплой.
- Проверь, что `payout` в записях соответствует реальным ставкам CPA — по ним считается выручка.
